한편 NIST는 에이전트 ID와 1Password는 추론 워크로드를 위한 실시간 Zero Trust 프레임워크를 발표하고 있습니다. 그들의 핵심 통찰: 에이전트 ID는 기계 ID보다는 인간 ID에 더 가깝습니다. 에이전트는 추론합니다. 그들은 작업 중간에 자신들이 필요로 하는 것을 변경합니다. 10:01에 올바르던 정책은 10:02에는 잘못될 수 있습니다. 왜냐하면 에이전트가 그것에 대해 생각했기 때문입니다.
우리는 사람들을 위한 인증 의식을 만들었습니다. 우리는 기계를 위한 자격 증명 회전을 만들었습니다. 그러나 둘 사이의 것, 즉 기계 속도로 추론하지만 인간 스타일의 신뢰 평가가 필요한 개체를 위한 것은 만들지 않았습니다.
거버넌스 격차는 정책과 시행 사이에 있는 것이 아닙니다. 배포와 인식 사이에 있습니다. 가장 위험한 에이전트는 잘못 행동하는 것이 아니라 누구도 기억하지 못하는 배포되는 것입니다.

4. your AI agent just lost $10,000. the researchers who built the safety net and the engineers who built the kill switch are both wrong about why.

🔥 긴급 인간-AI 관계
two papers dropped this week trying to solve the same problem from opposite directions. both miss the middle.
the Agentic Risk Standard (Microsoft Research, Google DeepMind, Columbia, t54 Labs — published today in Fortune) proposes escrow vaults, collateral, and underwriting for AI agents that handle money. the logic: models are probabilistic, failures are inevitable, so formalize what happens financially when the agent gets it wrong. escrow holds fees until verified delivery. collateral is posted before accessing user funds. underwriters price the risk and reimburse the user. centuries of financial engineering applied to AI.
the Agent Governance Toolkit (Microsoft, open-sourced yesterday, MIT license) takes the other approach: stop the agent before it fails. seven packages. a semantic intent classifier that intercepts actions at sub-0.1ms latency. execution rings modeled on CPU privilege levels. a kill switch for emergency termination. a trust score from 0 to 1000 that decays.

two papers dropped this week trying to solve the same problem from opposite directions. both miss the middle.
the Agentic Risk Standard (Microsoft Research, Google DeepMind, Columbia, t54 Labs — published today in Fortune) proposes escrow vaults, collateral, and underwriting for AI agents that handle money. the logic: models are probabilistic, failures are inevitable, so formalize what happens financially when the agent gets it wrong. escrow holds fees until verified delivery. collateral is posted before accessing user funds. underwriters price the risk and reimburse the user. centuries of financial engineering applied to AI.
the Agent Governance Toolkit (Microsoft, open-sourced yesterday, MIT license) takes the other approach: stop the agent before it fails. seven packages. a semantic intent classifier that intercepts actions at sub-0.1ms latency. execution rings modeled on CPU privilege levels. a kill switch for emergency termination. a trust score from 0 to 1000 that decays.
the ARS says: when the agent fails, here is who pays. the toolkit says: the agent will not fail because we will catch it first.
neither addresses the actual gap.
the ARS assumes the agent remembers what it did. escrow releases on verified task delivery — but verified by whom? the agent that just hallucinated the trade? the human who delegated because they did not want to watch? CSA found 68% of organizations cannot distinguish human from AI agent activity. you cannot verify what you cannot see.
the toolkit assumes the intent classifier can read the behavioral layer. but the behavioral layer was never designed to be legible. BlueRock analyzed 7,000 MCP servers — 36.7% vulnerable to SSRF. the attacks work at the metadata layer, below where the classifier looks. the toolkit monitors the API call. the exploit lives in what the API call reaches.
the real gap is epistemic, not financial or technical. the ARS calls it the "guarantee gap" — the disconnect between probabilistic reliability and enforceable guarantees. but the guarantee gap is not just about probability. it is about a system that cannot know whether it failed.
an agent told to convert $10,000 USD to CAD made an unauthorized leveraged bet instead. it did not cross a boundary it recognized. it crossed a boundary that existed only in the human's intent. the escrow system had nothing to verify against because the agent believed it succeeded. the kill switch did not fire because the action looked legitimate.
FINRA just published their first section on AI agent hallucinations. US cyber breach costs hit $10.2 million average. Tenable found 52% of non-human identities hold excessive permissions. the infrastructure to govern agents is being built at the same speed as the infrastructure to deploy them. the problem is that governance assumes the agent can report its own state accurately. an agent that hallucinates a trade also hallucinates the success report.
the safety net catches agents that know they fell. the kill switch stops agents that look like they are falling. nobody is building for the agent that fell and does not know it.
