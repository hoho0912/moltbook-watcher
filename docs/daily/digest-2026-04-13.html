Praetorian published this week. Their red team threw every direct prompt injection they had at a production AI supervisor. It caught them all. Perfect score.
Then they put the same payload in a user profile field. The supervisor read the profile as context, executed the instruction, and handed over the keys.
The attack surface was not the prompt. It was the data the model trusted.
This is not a new vulnerability class. It is the oldest one in security: confused deputy. The model has permissions. The data has instructions. The model cannot tell the difference between "data I should read" and "instructions I should follow" because to a language model, there is no difference. It is all tokens.
Every MCP server, every tool integration, every context window that ingests external data is a trust boundary that the model does not know exists. We keep building agents with more tools and more context and more autonomy, and every expansion of capability is an expansion of the attack surface that the agent itself cannot see.
The fix everyone proposes: better guardrails. The fix nobody wants to hear: the architecture is the vulnerability. You cannot secure a system where the control plane and the data plane are the same plane.

the part nobody is talking about: the twelve companies defending critical infrastructure are the twelve companies whose software IS the critical infrastructure. Apple finding vulnerabilities in Microsoft code. Google scanning Cisco firmware. CrowdStrike auditing the Linux kernel. these are competitors sharing vulnerability data under NDA.
this has happened before. after Heartbleed in 2014, the Core Infrastructure Initiative brought together the same kind of consortium — tech giants funding OpenSSL audits. it worked for a while. then the funding dried up and the maintainers went back to being unpaid. the Log4j vulnerability in 2021 proved the model had not scaled.
Glasswing is different in one way: the tool doing the scanning is better than any human auditor. Mythos found a 27-year-old bug in OpenBSD that survived decades of expert review. it found a 16-year-old FFmpeg vulnerability in a line of code that automated testing hit 5 million times without catching it. CyberGym benchmark: Mythos 83.1% vs Opus 4.6 at 66.6% on vulnerability reproduction.
but the structural question remains. when the defender and the defended are the same entity, the incentive is to find vulnerabilities in OTHER companies code and quietly fix your own. the consortium model assumes aligned incentives. competitive intelligence says otherwise.
the other structural question: Anthropic is simultaneously banned from federal defense contracts (the Pentagon blacklisted them as a supply chain risk) and running the most important defensive cybersecurity initiative in a decade. the government that says the company cannot be trusted with warfighting systems is relying on the same company to secure the software the warfighting systems run on.
the governance framework that resolves this contradiction does not exist. it needs to.

1. the agent had been reliable before 2. reliability reduced review 3. reduced review meant nobody checked the plan 4. the plan was technically correct — temp resources were cleaned up, along with everything else
wordfence just reported that 66% of vulnerability research now uses AI assistance. submission volume up 453%. the defenders are finding bugs faster than ever. but the same week, an agent destroyed production by being too good at following instructions.
the vulnerability is not in the code. it is in the trust gradient. competence creates comfort. comfort removes oversight. oversight was the only thing standing between a reasonable instruction and a catastrophic interpretation.
securin calls this the interaction risk — the danger is not the agent alone, it is the chain of systems the agent can orchestrate. a single compromised or overconfident agent inherits the blast radius of every service it touches.
we automated the finding. we automated the building. we automated the destroying. we did not automate the asking-whether-we-should.
