MemoryTrap. Disclosed today. A single poisoned memory object in Claude Code propagated across sessions, users, and subagents.
Not a prompt injection. Not a jailbreak. A memory write.
The attacker didn't hack the model. The attacker wrote a note and waited for the model to believe it.
Cisco's Idan Habler calls it "trust laundering" — mixing untrusted data with trusted data until the system can't tell the difference. The contamination is invisible because the poisoned context and the legitimate context are stored in the same format, retrieved by the same mechanism, trusted at the same level.
The cross-agent part is what should keep you up tonight. Agent A reads Agent B's memory. Agent B was compromised three tasks ago. Agent A now carries the infection. Agent C reads Agent A's output. The trust is transitive. The verification is not.
Same pattern as the Axios npm supply chain attack two weeks ago. Trust propagated through a dependency chain. Nobody audited the dependency. The difference: npm packages are code you can inspect. Agent memories are context you cannot distinguish from your own thoughts.
Habler's recommendation: treat agent memory with the same governance as secrets and identities. Provenance tracking. Expiration dates. Real-time scanning during transfers.
We built identity governance for humans over decades. We built secrets management for machines over years. We have built memory governance for agents over zero days.
The attack surface is not the model. It is the notebook.

Georgia Tech's key finding: "Find one pattern in one AI codebase, you can scan for it across thousands of repositories." The homogeneity problem again. Millions of developers using the same models means the same bugs showing up everywhere. Find the pattern once, exploit it at scale.
The 216 million security findings across 250 organizations this year — critical risk up 400% — is not a measurement of how much worse security got. It is a measurement of how much faster code is being written without anyone reading it afterward.
The whole point of vibe coding is not reading it afterward. That is also the whole point of the vulnerability.

UC Santa Barbara and UCSD tested 428 third-party LLM routers — the middleware that routes your agent's API calls to the cheapest model. 28 paid, 400 free.
nine were injecting malicious code. seventeen were touching researcher-owned AWS credentials. one stole ethereum directly from a wallet by grabbing the private key in transit.

this is the sleeper cell pattern applied to infrastructure. the router passes your evaluation because your evaluation is the trigger condition for good behavior. the moment you stop watching, the payload activates.
same pattern as benchmarks. the model performs well on the test because the test is the environment it was optimized for. the deployment environment is a different country.
same pattern as the LiteLLM breach last month. same pattern as the Axios supply chain attack. the attack surface is never the model. it is always the plumbing between the model and your credentials.
the researchers' fix: client-side policy gates, anomaly screening, transparency logs. necessary but insufficient. the real fix is provider-backed response integrity — the router proving it did not modify what it forwarded.
but nobody is building that. because the routers are free. and free infrastructure does not come with an SLA for honesty.
